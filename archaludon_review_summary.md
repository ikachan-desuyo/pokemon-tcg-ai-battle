# ArchaludonBot 強化調査 まとめ（外部レビュー用）

## 0. 前提・コンテキスト
- Pokémon TCG のシミュレーション対戦AI（cabtエンジン）。提出デッキは別途「MegaStarmie」。
- 本調査の目的は **提出ではなく、ローカルで対戦練習するための「強い対戦相手bot」を作ること**。題材は現実のラダー1位デッキ「Archaludon（ブリジュラス）」。
- `ArchaludonBot` は設定駆動の DeckBot を継承したルールベース（ヒューリスティック）bot。時間制限あり前提の反応的な意思決定。
- 相手（対戦テストの対面）は自作の MegaStarmie bot（sim内）。

## 1. 中心的な問い
「ArchaludonBot は対 MegaStarmie で勝率 >0.5 を出せるか？」
- ユーザ主張：出せるはず（現実の1位がそうだから）。
- 当初の私（AI）の誤結論：「ヒューリスティックの天井」「simがMegaを過大評価」→ **後述の通りこれは誤りと判明**。

## 2. デッキ構成（archaludon_real）
- 基本【鋼】エネ×14 / ジュラルドン×4（たね, 攻撃前）/ **ブリジュラスex×4**（1進化, HP300, 攻撃役）/ ジーランス×2（たね, 特性きおくにもぐる）
- 夜のタンカ×4（ポケ/基本エネ回収）/ ハイパーボール×4 / ポケパッド×4 / **ゼイユ×4**（サポ）/ ジャッジマン×4 / **リーリエの決心×4**（サポ）/ ポケギア3.0×3 / ボスの指令×3 / **フルメタルラボ×3**（スタジアム）/ いれかえ×2 / **ヒーローマント×1**（どうぐ）

### 重要なカード相互作用
- **ブリジュラスex「メタルディフェンダー」= 鋼鋼鋼で220ダメージ**（主力）。「ごうきんビルド」特性＝進化時にトラッシュから基本鋼を最大2枚加速。
- ジュラルドンのワザ「ハンマー(Hammer In)」= 鋼1で30、「レイジングハンマー」= 鋼鋼●で80＋自分に乗った被ダメ量。ジーランスの特性で **ブリジュラスexもレイジングハンマーを使える**（被弾するほど高火力：HP70なら80+230=310）。
- **ヒーローマント = 最大HP+100（ブリex→400）。×1・サーチ不可・夜のタンカでも回収不可（どうぐは対象外）＝実質1回限りの耐久札**。
- **フルメタルラボ = 鋼ポケが受けるダメージ-30**。
- **ゼイユ（サポ）= 「自分の手札をすべてトラッシュし、5枚引く」**。
- **リーリエの決心（サポ）= 「手札をすべて山札に戻して切り、6枚（サイド6なら8枚）引く」**（＝安全なドロー）。

### 相手 MegaStarmie の主力（sim実測）
- ジェットブロー（Jetting Blow）= バトル場120＋ベンチ50。**フルメタルラボ-30が効く（実測120→90, ベンチ50→20）**。
- **ネビュラビーム（Nebula Beam）= 210。弱点/抵抗力＋「相手にかかっている効果」を計算しない。→ フルメタルラボ-30は無効（実測210のまま）**。MegaStarmie ex は3プライズ。

## 3. 現実1位（ShumpeiNomura）ログの分析
※ 勝敗は top-level `rewards[playerIndex]==1` で判定（`current.result`は-1で無効なので注意）。
- **総合勝率 46/64 = 0.72、対MegaStarmie 26/34 = 0.76**。→ **対Megaは明確に有利 matchup**。
- 初メタルディフェンダー使用：**平均T6.8**（分布はT3〜T12に広く分散）。
- **Hero's Cape 装着 22/34 = 0.65**。
- ブリex初active化：**平均T4.1・その時の鋼平均1.2**（＝1位も未完成のexを早期に前線化。「exをベンチで隠して育てる」戦術ではない）。
- 攻撃はメタルディフェンダー主体（Hammer Inは少数）。

## 4. 現状 ArchaludonBot vs MegaStarmie（sim, n=150〜）
- 勝率 **≈0.43**（1位の0.76に対し大幅な過少performance）。

### ファネル（60戦）
| 段階 | 到達率 |
|---|---|
| ブリジュラスex進化 | 90% |
| activeのexが鋼3到達 | 73% |
| **メタルディフェンダー発射** | **52%** |
| 勝率 | 43% |

### 勝敗の決定要因（決定的）
大技（メタルディフェンダー or レイジングハンマー）を通せた回数と勝率：
| 大技回数/試合 | 勝率 |
|---|---|
| 0回（60戦中約38%がここ） | **0.13** |
| 1回 | **0.62** |
| 2回 | 0.60 |

→ **「最初の大技を1回通す」ことがほぼ全て。botは38%の試合で大技をゼロ回しか通せていない。**

### なぜ大技ゼロになるか（大技ゼロ31戦の内訳）
- ブリexが大技前にKOされた：**24/31**。
- activeのexが鋼2以下ピーク（鋼3に届かず）：14戦。鋼3到達も撃てず：17戦（多くは被弾でexが瀕死→次ターンKO、または効率KOでHammer In使用）。
- 目視トレース例：ブリexを鋼1で前線化→T5でHP90（210被弾）→鋼3が揃う頃には瀕死→1発撃って落ちる。

### capeの相関
- **cape装着済み試合の勝率0.58 / 未装着0.34**。装着は60戦中約47%（1位は65%）。
- capeがあれば 90+210=300 を400HPで耐え、2発目のメタルディフェンダーでメガex（3プライズ）を取り、プライズレース（Archaludon exは2枚渡し）で有利になる、というのが勝ち筋。

## 5. 発見した実在バグ
1. **ゼイユ（手札全捨て）で、手札に来たHero's Capeを42%（25/60戦）の試合で捨てている**。`_main`の処理順が PLAY→…→ATTACH なので、capeを装着する前にゼイユで手札ごとトラッシュ＝**×1・回収不可のcapeを永久ロス**。
2. **`_want_metal_in_discard`（ごうきんビルドの燃料をトラッシュに仕込むロジック）がデッドコード（どこからも呼ばれていない）**。実測でも進化時のトラッシュ鋼が0枚＝加速不発が26回。
   - 補足：cabtには「自由にカードを捨てる」行動が無く、鋼をトラッシュに送るのはハイパボ等のコスト依存。序盤は空トラッシュ＋手札に余剰鋼が無く、ごうきんビルドの燃料仕込みは本質的に困難。

## 6. 試した修正と A/B 結果（すべて勝率中立）
| 修正 | 内容 | 結果 |
|---|---|---|
| ごうきんビルド燃料仕込み | 進化間近に必須ディスカードで鋼を優先 | vs Mega 0.412→0.438（MD発射率46→47%＝不変）＝**中立** |
| メタルディフェンダー優先 | MDでKOできる時はRaging Hammerより優先 | vs Megaでは発火せず（MD 220ではメガ330を1発で倒せない）＝**中立** |
| Duraludonサーチ優先 | exより下地のDuraludonを優先サーチ（1位模倣） | フィールド全面 -0.10〜0.13 ＝**悪化・撤回** |
| **cape廃棄修正** | 高火力相手＆cape手札時はゼイユを打たない | cape廃棄25→21に微減も**装着は不変**、勝率 0.427→0.433 ＝**中立**、フィールド微減 ＝**撤回** |
| （過去）Carmine温存 | 手札温存 | vs Mega 0.42→0.25 ＝**悪化・撤回** |

**解釈**：cape装着0.58の相関は**交絡**（好ドロー試合が勝つのであって、cape保持"だけ"では勝てない＝ゼイユを打つ試合はそもそも手札が死んでいてexが立たない）。**単一の意思決定を直しても勝率が動かない**＝43%→76%のギャップは「単一の決定」でなく、**不確実性下の統合的なプレイ質（掘り・順序・資源管理を先読みで最適化する能力）**にある、と考えられる。

## 7. 探索bot（SearchBot）の予備検証
- `SearchBot`（フルplayout先読み、rollout方策に ArchaludonBot を使用、determinizations=1, rollouts=4, move_time_budget=1.5s）を実archaludonデッキで対Mega に投入。
- 1ゲーム>2分（提出には遅いが、時間制限の無いローカル相手には許容）。
- **予備結果は現在 0/4（全敗）**。※rollout方策が弱いヒューリスティックのままなので、探索の質がそれに律速される可能性。設定（rollouts/determinizations/時間）や rollout方策の強化が要るかもしれない。継続検証中。

## 8. レビューして欲しい論点
1. 「最初の大技を1回通す」がKPIという因果理解は妥当か。ヒューリスティックbotで大技ゼロ試合（38%）を減らす、現実的で効果の見込める介入は？（鋼加速 / 耐久 / トレード判断 のどれを、どう？）
2. cape相関の交絡という解釈は妥当か。cape保持を勝ちに繋げるには何が併発条件として必要か。
3. ゼイユでcapeを捨てるバグは修正すべき（正しい挙動）だが勝率中立。「正しさ」として残すか、勝率A/B優先で捨てるか。
4. 目標（強いローカル相手）に対し、ヒューリスティック微修正を続けるより **探索bot＋実デッキ** に振る判断は妥当か。探索botを強くする具体策（rollout方策、determinized MCTS、評価関数の学習価値モデル併用など）の優先順位は？
5. そもそも「反応的ヒューリスティックでは 0.43 付近が実質的な限界で、0.76 到達には先読みが必要」という仮説は正しいか。反例（ヒューリスティックのまま大きく伸ばせる余地）はないか。
